Page Rules

Page Rules provide you with fine-grained control of CloudFlare on each page of your site. Users can enter a URL pattern and then customize CloudFlare settings only for that pattern.
See the Invision prototype of this project.

Old version

The old version of the Page Rules UI required users to enter a URL pattern and then select a setting for every possible feature setting. This was not only confusing, but forced users to make settings that they may not want to make. It also did not offer a way for users to see whether no setting was set for a particular feature.

Process

We knew that we were going to keep much of the same functionality for Page Rules in the new UI. But we wanted to improve the usability of the app. I knew we needed to solve the problem in the old UI that forced users to provide a setting for every feature.

In addition, since we had redesigned our product dashboard, we now offered more CloudFlare features than we had when Page Rules was originally created. For example, CloudFlare now provides Multi-User functionality. For Page Rules, this means many different users could be contributing Page Rules to one website. I wanted to provide a way for users to share notes on Page Rules to help explain why they had created a specific rule.

Lastly, it was important to stop using an iframed version of an old UI in our new product dashboard. I wanted to redesign the application in our new visual style for consistency with the rest of the site. Because CloudFlare is a security and performance company, consistency is important to our design integrity and it helps establish trust with users.

Problem solving: Feature Selections

Rather than putting every feature in front of the user in a long-form format, I chose to design a cleaner UI that allows users to add one feature setting at a time. Users can either type to begin searching for the feature of their choice, or simply click to open our dropdown list to scroll through the different features available. The features are divided into categories, which reflect the same categories users navigate through in the primary dashboard UI.

Problem solving: Multiuser

To help users share information about different Page Rules, I included a Description option for all Page Rules. This feature is completely optional, and allows users to add brief notes about what a Page Rule does. The description is instantly visisble in the Page Rules table for easy skimming.

Problem solving: Staging Mode

Another feature that we wanted to implement as a result of multi-user was staging mode. Because multiple members of a team may be accessing the same site, it's important to provide users the ability to draft up a page rule, save it, and send it off for review to another team member before activating the Page Rule.

To accommodate for this, I designed a simple selection in the UI that allows users to save Page Rules as drafts. Upon saving, the Page Rule is added to the table, but is set in the 'off' position to prevent it from executing. When a user is ready to activate the Page Rule, they simply switch it from off to on.

In addition, whenever a new Page Rule is added to a site, when it appears in the table the row glows green for a few seconds, then appears normal. This helps users to locate the Page Rule they have just added, especially in cases where users may have up to 100 rows of Page Rules in their table.

Full Design

The finished design involves a simple table display that showcases Page Rules a user has created, in a ranked order of which Page Rule is most important and should take priority in execution. Page Rules can be dragged and dropped, or the rank can be clicked to edit right in the table. Users can continue to manage or delete Page Rules from the application. Once a user has used all of the available Page Rules for their plan level, they are encourage to upgrade their CloudFlare account to obtain more Page Rule slots.

For a more comphrensive and interactive look at this design, please see the Invision prototype of this project.
